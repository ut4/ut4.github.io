Reittien käyttäjäoikeudet

Tämä esimerkki demonstroi miten voit hylätä tai hyväksyä HTTP-pyyntöjä käyttäjän roolin perusteella käyttämällä Pike\Auth\ACL-luokkaa, reittimiddlewarea, ja reittikonteksteja.

Esimerkin koodi löytyy kokonaisuudessaan github.com/ut4/pike/tree/master/examples-kansiosta.

Step 2: Reittimiddlewaren implementointi

Katso myös. Käyttäjäroolit ja oikeudet.

AuthorizingRoutes/MyAuthModule.php

...

abstract class MyAuthModule {
    /**
     * @param \stdClass $ctx {\Pike\Router router}
     */
    public static function init(\stdClass $ctx) {
        $ctx->acl = new \Pike\Auth\ACL;
        $ctx->acl->setRules(self::makeMyAclRules());
        //
        $ctx->router->on('*', function ($req, $res, $next) use ($ctx) {
            // Tämä tulisi normaalisti sessiosta ($ctx->auth->getIdentity()->role)
            $userRole = LOGGED_IN_USER_ROLE;
            // ks. Step 3
            [$action, $resource] = explode(':', $req->routeInfo->myCtx);
            if (!$ctx->acl->can($userRole, $action, $resource))
                $res->status(403)->json(['err' => 'Not permitted']);
            else
                $next();
        });
    }
    /**
     * @return \stdClass
     */
    private static function makeMyAclRules(): \stdClass {
        // Nämä tulisi normaalisti esim. tiedostosta tai tietokannasta.
        $resources = (object) [
            'products' => ...,
            'reviews' => ...
        ];
        $userPermissions = (object) [
            ACL::ROLE_EDITOR => (object) [
                'products' => ...,
                'reviews'  => ...,
            ],
            ACL::ROLE_CONTRIBUTOR => (object) [
                'products' => ...,
                'reviews'  => ...,
            ]
        ];
        return (object) [
            'resources' => $resources,
            'userPermissions' => $userPermissions
        ];
    }
}

Step 3: ACL-informaation lisääminen reitteihin

  • AuthorizingRoutes/Review/ReviewModule.php

abstract class ProductModule {
    /**
     * @param \stdClass $ctx {\Pike\Router router}
     */
    public static function init(\stdClass $ctx) {
        $ctx->router->map('POST', '/products',
            [ProductController::class, 'handleCreateProduct', 'create:products']
        );
        $ctx->router->map('PUT', '/products/[i:productId]',
            [ProductController::class, 'handleEditProduct', 'edit:products']
        );
        $ctx->router->map('POST', '/products/[i:productId]/comment',
            [ProductController::class, 'handleAddComment', 'comment:products']
        );
    }
}

Testipyynnöt ja vastaukset

Esimerkin applikaatiota voi testata esimerkiksi php:hen bundlatulla dev-serverillä:

  1. Aja cd examples
  2. Aja php -S localhost:8080
  3. Päivitä authorizing-routers.php -tiedoston define('LOGGED_IN_USER_ROLE', <arvo>)

Roolille ACL::ROLE_CONTRIBUTOR 

POST 'http://localhost:8080/authorizing-routes.php?q=/products'
-> '{"err":"Not permitted"}'

POST 'http://localhost:8080/authorizing-routes.php?q=/products/1/comment'
-> '{"ok":"ok"}'

PUT 'http://localhost:8080/authorizing-routes.php?q=/products/1'
-> '{"err":"Not permitted"}'

POST 'http://localhost:8080/authorizing-routes.php?q=/reviews'
-> '{"insertId":1}'

PUT 'http://localhost:8080/authorizing-routes.php?q=/reviews/1/approve-or-reject'
-> '{"ok":"ok"}'

Roolille ACL::ROLE_AUTHOR 

POST 'http://localhost:8080/authorizing-routes.php?q=/products'
-> '{"err":"Not permitted"}'

POST 'http://localhost:8080/authorizing-routes.php?q=/products/1/comment'
-> '{"ok":"ok"}'

PUT 'http://localhost:8080/authorizing-routes.php?q=/products/1'
-> '{"ok":"ok"}'

POST 'http://localhost:8080/authorizing-routes.php?q=/reviews'
-> '{"insertId":1}'

PUT 'http://localhost:8080/authorizing-routes.php?q=/reviews/1/approve-or-reject'
